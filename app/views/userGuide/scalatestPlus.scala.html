ScalaTest +

ScalaTest is friendly and does not stand alone, we understand that ScalaTest needs to integrate and play well with other frameworks like Play to increase developer productivity.

ScalaTest + Play

ScalaTest + Play is an integration library between ScalaTest and Play Framework. It provides useful API to help developer to write tests for Play application efficiently using ScalaTest.

ScalaTest + Play can be used in current 2.2.x version of Play, to use it just add the following dependency in your build.sbt/Build.scala: 

libraryDependencies += "org.scalatestplus" %% "play" % "1.0.0" % "test"

The following is a quick summary of main traits and classes provided by ScalaTest + Play to help you Play-Testing your application:

Name Description
PlaySpec Convenience base class for Play tests.
FixedPlaySpec Convenience base class for "mixed fixture" Play tests.
OneAppPerSuite Provides a new FakeApplication instance per suite.
OneAppPerTest Provides a new FakeApplication instance for each test.
OneServerPerSuite Provides a new FakeApplication and running TestServer instance per suite.
OneServerPerTest Provides a new FakeApplication and running TestServer instance for each test in suite.
OneBrowserPerSuite Provides a new Selenium WebDriver instance per suite.
OneBrowserPerTest. Provides a new Selenium WebDriver instance for each test in suite.
AllBrowsersPerSuite Run suites of tests using multiple browsers available on the running system, all tests in a suite share the same instance of browser.
AllBrowsersPerTest Run suites of tests using multiple browsers available on the running system, all tests will get their own instance of browser.
ConfiguredApp Provides a configured FakeApplication (througn config map) to the suite.
ConfiguredServer Provides a configured FakeApplication and server port number (through config map) to the suite.
ConfiguredBrowser Provides a configured FakeApplication, server port number, and Selenium WebDriver (through config map) to the suite.
WsScalaTestClient Provides convenience methods to create WS requests in tests, already mixed into PlaySpec and MixedPlaySpec.
